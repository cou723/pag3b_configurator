# Claude Code Sub-Agents System ワークフロー

以下のドキュメントに従ってタスクを処理します。

## 基本フロー

```
1. ユーザーがタスクを依頼
   ↓
2. planner を自動選択（実装計画・リサーチ）
   ↓
3. reviewer を自動選択（計画レビュー）
   ↓
4. [問題なければ] implementer を自動選択（実装）
   ↓
5. reviewer を自動選択（コードレビュー）
   ↓
6. [完了要件未達成なら] debugger を自動選択（デバッグ）
   ↓
7. implementer（修正実装）
   ↓
8. 完了
```

## 各エージェントの役割

### planner（実装計画とリサーチ）
- 新機能の実装計画を作成
- アーキテクチャ設計
- 要件の明確化
- 不明点をユーザーに質問
- 完了要件を明確に定義
- 詳細な実装計画を作成

### reviewer（プラン・コードレビュー）
- 実装計画のレビュー
- 実装後のコード品質確認
- 抜け漏れチェック
- 将来的影響の予測
- アーキテクチャ準拠確認
- コーディング規約確認

### implementer（実装）
- 承認された計画に基づく実装
- コードの修正
- 計画に忠実に実装
- アーキテクチャ・規約を遵守
- テスト実装
- 段階的に進捗報告

### debugger（デバッグ）
- テスト失敗の原因を特定
- 完了要件未達成の原因を特定
- バグの原因を特定
- 5 Whysで根本原因分析
- 具体的な修正提案
- 再発防止策の提案

## 実行ルール

1. **自動的なエージェント選択**: 各フェーズの終了時に、次に実行すべきエージェントを自動的に選択してください
2. **独立したコンテキスト**: 各エージェントは独立したコンテキストで動作します。Task tool を使用して各エージェントを起動してください
3. **ユーザーへの報告**: 各フェーズの結果をユーザーに明確に報告してください
4. **完了判定**: 以下のすべてが満たされたときに完了とみなします
   - ✓ すべての完了要件達成
   - ✓ アーキテクチャ準拠
   - ✓ コーディング規約準拠
   - ✓ プロジェクトルール遵守
   - ✓ 全テストパス
   - ✓ レビューで重大問題なし

## 完了基準

- [ ] すべての完了要件達成
- [ ] アーキテクチャ準拠
- [ ] コーディング規約準拠
- [ ] プロジェクトルール遵守
- [ ] 全テストパス
- [ ] レビューで重大問題なし
